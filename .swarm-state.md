## Iteration 1 — Replace audio stubs + add AudioDiagnosticsOverlay

### What changed
- Replaced all 23 silent stub MP3s in `public/sounds/` with valid 1-second ffmpeg-generated sine tones:
  * Ambient files (~110 Hz), footsteps (~220 Hz), collisions/landings (~180 Hz)
  * Jumps (~660 Hz), splashes (~880 Hz), paddle/boost (~440 Hz)
- Created `src/components/AudioDiagnosticsOverlay.tsx`:
  * DEV-gated (`import.meta.env.DEV` check at top of component)
  * Renders as a fixed-position HTML div overlay (mounted inside existing `<Html>`)
  * Surfaces AudioContext state, buffer load count (loaded/total/failed), active sounds with elapsed time, segment ambient phase, and reactive crossfade volumes
  * Updates every 500 ms via `setInterval` (no RAF budget)
- Updated `src/systems/AudioSystem.ts`:
  * Added `failedSounds` tracking in `loadSound`
  * Added `getLoadStatus()`, `getActiveSounds()`, `getAudioContextState()`, `setReactiveVolumes()`, `getReactiveVolumes()`
  * Wired `audioContext` to `listener.context` for state introspection
- Updated `src/Experience.jsx`:
  * Imported `AudioDiagnosticsOverlay`
  * Mounted it inside the existing `<Html>` overlay block, gated with `{import.meta.env.DEV && <AudioDiagnosticsOverlay />}`
