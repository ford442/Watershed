## Iteration 2 — Buoyancy integration

### What changed
- `src/components/Environment/FloatingObjectManager.tsx` (task description listed `src/systems/FloatingObjectManager.ts` but the source lives in `components/Environment`):
  * Imported `getWasm` and `WatershedNativeModule` from `../../systems/WatershedWasm`
  * Imported `WATER_DENSITY` and `GRAVITY` from `../../constants/game`
  * Removed now-unused `calculateBuoyancyForce` import
  * Added module-level WASM lazy-load: `let wasmModule: WatershedNativeModule | null = null; getWasm().then(m => { wasmModule = m; }).catch(() => {});`
  * Replaced the `calculateBuoyancyForce(...)` call with:
    ```ts
    const submergedVolume = FLOATING_OBJECT.DEBRIS_VOLUME * submergedRatio;
    const buoyancyForce = wasmModule
      ? wasmModule.computeBuoyancy(submergedVolume, WATER_DENSITY, GRAVITY)
      : submergedVolume * WATER_DENSITY * GRAVITY; // JS fallback
    if (isFinite(buoyancyForce)) {
      api.applyImpulse({ x: 0, y: buoyancyForce * dt * PHYSICS_SCALE, z: 0 }, true);
    }
    ```
- `src/systems/WatershedWasm.ts` (allowed file):
  * Replaced the bare `import('/watershed_native.js')` with a `new Function('url', 'return import(url)')` wrapper so Vite/Rollup does not attempt to resolve the Emscripten glue JS at build time (the file is only present after `npm run build:wasm`).
